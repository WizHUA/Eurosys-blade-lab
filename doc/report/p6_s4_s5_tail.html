5. 消融实验设计

论文贡献点必须通过消融实验量化验证。以下 7 组实验覆盖全部核心设计维度，每组均在相同的 29 个 formaltest 场景上运行，使用相同的 evaluation harness（eval/evaluate.py）。

编号 实验名称 消融方式 验证的研究问题 论文 Contribution
Full 完整系统（v4） 无消融 完整系统性能上限 —
Abl-A 无 Triage 跳过 Stage 1；将所有指标的统计摘要直接输入 HYPOTHESIZE 确定性分诊对准确率和 token 效率的贡献 Hybrid Control
Abl-B 无 Evidence Gating 移除 GATE 节点；THINK 节点输出 conclude 时直接 FINALIZE Evidence Gating 对误报率和幻觉率的抑制效果 Evidence Gating

Abl-D 无时序因果先验 FocusContext 的 causal_order 打乱顺序（随机排列 Top-K 指标） 时序因果排序对 HYPOTHESIZE 质量的影响（v4 特有贡献，相比 v3） Temporal Causal Prior
